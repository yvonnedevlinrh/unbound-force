## Why

The `/finale` command's PR description generation (Step 5c)
produces a minimal `## Summary` with bullet points from commit
history. This is underspecified compared to sibling commands
(`/unleash` Step 8, `/review-pr` attribution). The current
state has several gaps:

1. **No structured sections** — reviewers lack "how to test"
   and "how to demo" guidance, forcing them to reverse-engineer
   verification steps from commit diffs.
2. **No AI attribution** — PR descriptions and commit messages
   generated by `/finale` lack provenance indicators, making
   it impossible to distinguish AI-generated content from
   human-written content.
3. **No review-council findings** — unresolved findings from
   `/review-council` are lost when the PR is created, leaving
   reviewers without context on known issues.
4. **No PR template detection** — `/finale` ignores repository
   PR templates (`.github/PULL_REQUEST_TEMPLATE.md`), producing
   inconsistent PR formats across repos that use templates.
5. **Shell injection vector** — the current `gh pr create`
   call uses inline `--title` and `--body` shell interpolation,
   which is unsafe when AI-generated content contains shell
   metacharacters.
6. **Inconsistent AI attribution across commands** — three
   commands produce commit messages with AI attribution but
   use different trailer keys (`AI-assisted-by` vs
   `Assisted-by`), different value formats (command name vs
   tool name), and inconsistent model identification.
   `/address-feedback` and `/review-pr` have a `<model>`
   placeholder that is never resolved to the actual model.
7. **No model identification** — commit message attribution
   does not identify which AI model produced the code,
   making it impossible to correlate code quality with
   specific models or trace provenance beyond "AI-assisted."

Addresses: #142

## What Changes

Enhance `/finale` Step 5 (PR creation) and Step 3 (commit
message generation) to produce richer, safer, and more
transparent output. Standardize AI attribution across
`/finale`, `/address-feedback`, and `/review-pr` to use a
consistent trailer format that includes the model name.

## Capabilities

### New Capabilities
- `structured-pr-body`: Generate PR body with structured
  sections (What Was Done, How to Test, How to Demo, Key
  Files Changed) mirroring `/unleash` Step 8 patterns
- `pr-attribution`: Add AI attribution footer to PR
  descriptions: `_This PR was generated by /finale
  (AI-assisted)._`
- `commit-attribution`: Add AI attribution to commit
  messages via both a git trailer (`Assisted-by:
  <model>`) and a human-readable footer line
- `model-attribution`: Include the AI model name in
  commit message attribution trailers across all
  commands that generate commits (`/finale`,
  `/address-feedback`, `/review-pr`)
- `cross-command-consistency`: Standardize the
  attribution trailer key and value format across
  `/finale`, `/address-feedback`, and `/review-pr`
- `review-council-findings`: Surface unresolved
  `/review-council` findings in a "Known Issues" section
  of the PR body
- `pr-template-detection`: Detect and respect
  `.github/PULL_REQUEST_TEMPLATE.md` when present,
  mapping generated content to template sections

### Modified Capabilities
- `pr-creation`: Switch `gh pr create --body` to
  `--body-file` pattern to eliminate shell injection
  vector (consistent with `/review-pr`'s safe pattern)
- `commit-message-generation`: Extend Step 3 to append
  AI attribution trailer and footer to generated commit
  messages, using standardized format with model name

### Removed Capabilities
- None
